Para o site como ele está hoje, **não precisa de Supabase**.

O cardápio é estático — os produtos estão no HTML, a lógica no JS, e o pedido vai direto pro WhatsApp. Funciona perfeitamente sem banco de dados.

**Supabase só valeria a pena se no futuro quiserem:**

- Painel admin para o dono editar produtos, preços e fotos sem mexer no código
- Histórico de pedidos salvos
- Relatórios de vendas
- Cupons de desconto
- Controle de estoque

**Por enquanto o fluxo é simples e eficiente:**

Cliente escolhe → monta carrinho → manda pro WhatsApp → dono atende. Sem complicação, sem servidor, sem banco.

Se o cliente crescer e pedir um painel pra editar o cardápio sozinho, aí sim vale entrar com Supabase — e seria uma nova versão do projeto com valor maior pra cobrar. 🦈 Boa pergunta! Depende de quais páginas você quer adicionar.

**Páginas que NÃO precisam de Supabase:**
- Página de contato (só abre WhatsApp)
- Página "Sobre nós"
- Galeria de fotos da loja
- Promoções fixas

Essas são só HTML/CSS a mais, sem banco de dados.

**Páginas que AÍ SIM precisariam de Supabase:**
- Painel admin — dono edita cardápio, preços e fotos sozinho
- Cadastro de clientes e fidelidade (ex: "compre 10 ganhe 1")
- Histórico e relatório de pedidos
- Sistema de avaliações/comentários
- Cupons de desconto dinâmicos

---

**Como ficaria o valor com Supabase:**

Se for só páginas estáticas extras — **R$ 800 a R$ 1.200** nos próximos clientes, sem Supabase.

Se entrar painel admin com Supabase — **R$ 2.000 a R$ 3.500**, porque aí vira um sistema de verdade com autenticação, banco de dados, CRUD de produtos. É outro nível de projeto.
